# 4GBLE093 固件 TCP 通信协议分析

> 提取自固件源码 `Source Code_4GBLE093_V10_260227.zip`
> 当前固件连接: 159.75.109.42:8088 / 159.75.109.42:18088
> **要求: 固件不能改，服务器侧加 TCP Server 桥接到平台 API**

---

## 一、通信方式

- **协议**: TCP 透传（AT 命令 `AT+QIOPEN` 打开 TCP socket）
- **IP/端口**: 保存在 EEPROM `flash_save_t` 中，可通过 BLE/APP 远程修改
- **IP 存储**: 4 字节 BCD（如 159.75.109.42 存为 `0x9F,0x4B,0x6D,0x2A`）
- **端口存储**: 2 字节 Big-Endian

```
flash_save_t 结构体:
  u8 head[2];           // 校验头
  lock_id_t lock_id;    // 锁 ID (SN)
  u8 key_code[6];       // 密钥码
  u8 IP[4];             // 服务器 IP (159.75.109.42)
  u16 port;             // 端口 (8088 / 18088)
```

**锁连接后立即发送 logon 帧，然后按 interval 定时上报心跳 + GPS。**

---

## 二、数据帧格式（锁 → 服务器）

```
偏移  字段         长度   说明
[0]   FrameStart   1     固定 0xFE
[1-4] LockSN       4     锁序列号 (32-bit)  
[5]   Addr         1     地址域
[6]   Sub          1     功能码
[7]   SubLen       1     子功能长度
[8]   DataLen      1     数据长度 (= 包体总字节数 - 5)
[9+]  Payload      N     数据体
[-2]  CRC          1     CRC8 (从 Addr 到 Payload 末尾)
[-1]  End          1     固定 0xFF
```

**注意**: `DataLen` 是从 `Addr`(offset 5) 到 Payload 末尾的总长度。Payload 起始于 `DataLen+1` 的位置（offset 9）。

## 三、锁上报的功能码

### 1. Sub=0x01: 登录注册帧 (load_logon)
```
Addr=0x07, Sub=0x01, SubLen=0x00, DataLen=0x2E
Payload:
  [0-13]  GPS 定位数据 (14字节，load_location_message)
  [14-19] BLE MAC 地址 (6字节)
  [20-23] 当前服务器 IP (4字节)
  [24-25] 当前服务器端口 (2字节)
```
上报时机: TCP 连接建立后立即发送

### 2. Sub=0x06: 心跳帧 (load_heartbeat)
```
Addr=0x05, Sub=0x06, SubLen=0x00
Payload: 固件版本号字符串 (soft_ver)
```
上报时机: 周期性心跳

### 3. Sub=0x0A: GPS 定位帧 (load_GPSdata)
```
Addr=0x03, Sub=0x0A, SubLen=0x00, DataLen=0x29
Payload:
  [0-3]   时间戳 (4字节, Unix Timestamp)
  [4-7]   纬度 (4字节 BCD, ddmm.mmmm)
  [8-11]  经度 (4字节 BCD, dddmm.mmmm)
  [12]    速度 (1字节, 0x00-0x7F)
  [13]    方向标志位 (D7=北纬0/南纬1, D6=东经0/西经1)
  [14-...] 锁状态 + 电池电量等
```
上报时机: 周期性 GPS 上报

### 4. Sub=0x2D: 施封/解封/上锁事件帧
```
Addr=0x03, Sub=0x2D, SubLen=0x04
Payload:
  [0]     业务头 0x2A
  [1]     重复标志 0x55
  [2]     Cmd: 0x80=施封 / 0xA0=解封 / 0x62=上锁
  [3-6]   LockID (4字节)
  [7]     Gate (1字节)
  [8]     电池电量 (1字节)
  [9]     锁状态 (1字节: 0x50=关, 0x30=开)
  [10]    电机状态 (1字节)
  [11-14] 角度/位置 (4字节)
  ...
```
上报时机: 锁状态变化时

### 5. Sub=0x16: 确认回执 (load_reportseal / reportsenddelay / reportsleepset)
```
Addr=0x03, Sub=0x16, SubLen=0x03, DataLen=0x08
Payload:
  [0]     固定 0x2D
  [1-2]   report_Serial (服务器下发的流水号，回传)
```
上报时机: 收到服务器指令后回执

## 四、服务器下发指令（服务器 → 锁）

帧头同样以 0xFE 开头，锁 SN 在 [1-4] 字节。根据 [1] 字节分为 4 种命令类型:

### CMD 0x21 — 配置类
| [2] | 功能 | Payload |
|-----|------|---------|
| 0x10 | 设置时间 | 时间字符串 `YY/MM/DD,HH:MM:SS+TZ` |
| 0x03 | 设置上报间隔 | [5-6]=间隔(秒), [8-9]=流水号 |

### CMD 0x31 — 改 IP/端口
```
格式: "IP地址,端口" (ASCII 字符串)
如: 120.77.218.138,80 → ASCII 编码
锁收到后写入 EEPROM 并回复 setipOK
```

### CMD 0x51 — 休眠配置
| [2] | 功能 | Payload |
|-----|------|---------|
| 0x11 | 设置休眠时间 | `TT%SLEEP` + 3位数字(秒), [18-19]=流水号 |

### CMD 0x81 — 锁操作
| [2] | [7] | 功能 | 说明 |
|-----|-----|------|------|
| 0x2D | 0x12 | 查询锁状态 | 返回上锁/开锁状态 |
| 0x2D | 0x32 | 施封 | 封印/加锁 |
| 0x2D | 0xA0 | 开锁 | [23-28]=开锁密码(6位), [29]=有效时间(分钟) |

## 五、需要构建的 TCP 桥接服务

### 基本功能
1. 在 `120.77.218.138:8088` 监听 TCP 连接
2. 解析锁的二进制帧（0xFE...0xFF）
3. 根据 Sub 字段路由到平台 API

### 消息路由映射
| 锁上报 | 桥接动作 |
|--------|----------|
| Sub=0x01 登录帧 | 识别 lockSN → 查询设备 → 记录上线 |
| Sub=0x06 心跳 | 更新设备心跳时间 |
| Sub=0x0A GPS | 写入设备位置 (经纬度+速度) |
| Sub=0x2D 事件 | 写入操作日志 (施封/解封/上锁) |
| Sub=0x16 回执 | 关联服务器下发的流水号 |

### 下行指令路由
| 平台 API | 桥接动作 |
|----------|----------|
| 开锁 | CMD 0x81 → 0x2D+0xA0 (带密码和有效时间) |
| 查询状态 | CMD 0x81 → 0x2D+0x12 |
| 改上报间隔 | CMD 0x21 → 0x03 |
| 设置时间 | CMD 0x21 → 0x10 |

### 注意
- CRC8 校验算法需要从固件中提取 `get_CRC8()` 函数
- 每个下行指令需要生成唯一的 report_Serial 流水号
- TCP 断线重连: 锁会自动重连
- 多设备管理: 每个 TCP 连接对应一个设备 ID

## 六、依赖关系

- **端口 8088**: 数据通道（锁上报 + 服务器下发指令）
- **端口 18088**: 同上（可能是不同协议版本或开发环境用的）

服务端需要同时监听这两个端口。
