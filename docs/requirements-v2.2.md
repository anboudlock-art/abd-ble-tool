# 第一部分：设备注册与绑定

## 1.1 锁号方案：8位数字

锁号 = 8位数字。年份后1位 + 月份2位 + 批次流水号5位。

| 位 | 1 | 2-3 | 4-8 |
|---|---|---|---|
| 含义 | 年份后1位 | 月份 | 流水号 |
| 示例 | 6 | 05 | 00001 |

- 2026年5月第1台 → `60500001`
- 2026年5月第88台 → `60500088`
- 每月流水号从00001自动递增

## 1.2 核心原理

所有锁都有蓝牙。注册 = 扫QR + BLE采MAC+IMEI + 上传绑定。

| 设备类型 | BLE采MAC | BLE采IMEI |
|---|---|---|
| BLE 挂锁 | ✅ | 无 |
| 4G 挂锁 | ✅ | ✅（BLE同时读） |
| 4G 铅封 | ✅ | ✅ |

## 1.3 自动注册（三步）

1. **扫QR拿锁号** — APP摄像头扫锁身QR，识别锁号+型号，确认下一步
2. **BLE采集MAC+IMEI** — APP自动开蓝牙，搜索ABD前缀设备，选信号最强连接，一次性采集MAC+IMEI+固件
3. **确认上传入库** — 显示锁号/MAC/IMEI/型号/固件/新生产库，确认后平台自动创建设备记录、绑定MAC/IMEI、入库新生产库

## 1.4 手动注册（保底）

BLE故障时手输锁号+MAC+IMEI。

**异常处理：**

| 异常 | 处理 |
|---|---|
| QR扫不出 | 手动输入锁号 |
| BLE搜不到 | 检查上电→靠近重扫→切手动 |
| BLE连不上 | 重试3次→切手动 |
| IMEI读不到 | 可跳过，后续补采 |
| 锁号已存在 | 提示"已注册" |

# 第二部分：生产测试与报告

## 2.1 12项测试

**自动测试（6项）：**

| # | 测试项 | 标准 |
|---|---|---|
| 1 | BLE通信 | >-90dBm、<500ms |
| 2 | 4G联网 | 成功、<5s |
| 3 | GPS定位 | 搜星≥4、<30s |
| 4 | 电池电压 | 3.6V-4.2V |
| 5 | 功耗 | 待机<0.5mA |
| 6 | 固件版本 | 匹配预设 |

**环境测试（2项）：** 高低温 -40~+85℃各2h、防水IP67 30min

**人工检查（4项）：** 锁体开关10次、外观、指示灯、配件

## 2.2 测试APP

自动测试页逐项显示结果，底部 [❌不合格] [→下一步]。环境+人工页勾选+测试员信息，[✅合格] [❌不合格]。

## 2.3 导出报告

按批次选择，汇总统计+明细。PDF（含公章位）或Excel导出。

## 2.4 测试员角色

`production_operator` — 只能操作生产APP，不能登录PC管理后台。

# 第三部分：设备流转与授权管理

> **核心规则**：班组只是组织分组，授权必须到具体人员。设备没有授权人就不可用。

## 3.1 生命全周期

```
注册→测试→新生产→移入→待移交→发货→客户仓库→分配班组→授权到人→部署→使用→退回维修
                    ↑                          ↑
              APP申请审批←──────────APP临时开锁审批
```

## 3.2 厂商设备库存（三库）

### 总览

三张卡片并排：📦待移交 N台 | 🏭新生产 N台 | 🔧维修中 N台

### 🏭 新生产库

测试中的设备。测试通过后点 [📦 移入待移交] → 弹窗列出所有✅通过的设备 → 全选 → 确认一次性移入待移交库。

### 📦 待移交库

已测试通过，按批次分组展示。有设备列表、[全选本批]、[🚚 发货到公司]、[📊 导出测试报告]、[🔧 退回维修]。

**发货弹窗**分两步：
1. 选择设备（按批次分组，可全选某批或勾选部分）
2. 选择目标客户公司 + 发货单号 + 备注

> 关键逻辑：一批待移交设备可分批发给不同公司。每次发货选一个客户+若干设备。

### 🔧 维修中库

退回返修的设备，含退回公司、故障原因、维修状态。底部 [🔄 重新测试]（已修好→新生产）、[🗑 报废]、[📋 导出维修记录]。

### 三库流转汇总

```
新生产 →（移入待移交）→ 待移交 →（发货）→ 客户仓库
  ↑                      │                    │
  │ 重新测试              │ 退回维修            │ 退回
  ↓                      ↓                    ↓
维修中 ←──────────────── 故障退厂 ←─────────────┘
```

| 按钮 | 位置 | 弹窗列表 | 目标 |
|---|---|---|---|
| [📦 移入待移交] | 新生产库 | ✅通过的设备 | 待移交库 |
| [🔧 退回维修] | 新生产/待移交 | 需返修的设备 | 维修中库 |
| [🚚 发货到公司] | 待移交库 | 待移交设备(分批)+选客户 | 客户公司 |
| [🔄 重新测试] | 维修中库 | ✅已修好的设备 | 新生产库 |
| [🗑 报废] | 维修中库 | 任何状态 | 退役 |

## 3.3 公司设备仓库

客户公司收到设备后，"未分配"卡片显示"←新到！"。设备汇总卡片：各班组卡片 + 未分配卡片。

公司设备列表表格，底部：[确认入库] [分配到班组] [批量授权]。

## 3.4 分配班组弹窗

已选设备 + 选部门 + 选班组。提示：只是班组归属，开锁仍需授权到人。

## 3.5 批量授权（N×M）

三步弹窗：
1. 确认设备（多选N台）
2. 选人员（多选M人）
3. 预览N×M条授权，设有效期

## 3.6 现场部署弹窗

选场景类型（固定安防→门号 / 物流→车牌 / 危化品→仓库码 / 跨境→集装箱号）→ 填位置 → GPS → 拍照。部署后设备状态"运行中"。

## 3.7 授权管理 `/authorizations`

表格：设备号/位置/班组/人员/有效期/状态。✅有效 ⚠️将过期 ❌过期。[撤销] [更换授权人]

## 3.8 设备详情页

设备号/MAC/状态/型号/位置/电量/公司/班组。授权卡片含授权人/有效期，可撤销/更换。[开锁][关锁]

## 3.9 APP 申请开锁权限（长期）

| | ① 申请开锁权限 | ② 临开审批 |
|---|---|---|
| 场景 | 长期授权 | 临时开锁 |
| 时效 | 长期 | 1h/2h/4h/8h到期 |
| 设备 | 可多台 | 单台 |
| 审批页 | /permission-approvals | /temporary-approvals |

**APP端：** 选多台设备→填事由→提交→待审批/已通过/已拒绝三tab

**PC审批 `/permission-approvals`：** 表格+审批弹窗。可全同意/部分同意/拒绝。

## 3.10 APP 临开审批

**APP端：** 单台设备+选时长+事由+🔴紧急标记

**PC审批 `/temporary-approvals`：** 待审批/使用中/已过期三tab，🔴优先。使用中显示倒计时，可随时撤销。

## 3.11 审批规则

1. 长期可全同意/部分同意/拒绝
2. 临开到期自动失效
3. 审批人自动匹配：组长→部门→公司
4. 🔴紧急优先提醒
5. 记录可查不可删
6. APP/PC双端同步
7. 管理员可随时撤销临开

## 3.12 账号角色

| 角色 | 权限 | 操作 |
|---|---|---|
| vendor_admin | 厂商全局 | 三库管理、发货、导出报告 |
| production_operator | 生产 | 设备注册、出厂测试（仅APP） |
| company_admin | 本公司 | 入库、分配、授权、审批 |
| dept_admin | 部门 | 管理班组 |
| team_leader | 班组 | 班组设备、部署、审批 |
| member | 成员 | APP开锁、申请权限/临开 |
